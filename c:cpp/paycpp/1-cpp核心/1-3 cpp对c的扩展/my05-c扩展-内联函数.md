# 内联函数(inline function)

效率~


第一个在c中也会出现，宏看起来像一个函数调用，但是会有隐藏一些难以发现的错误。
第二个问题是`C++`特有的，预处理器不允许访问类的成员，也就是说预处理器宏不能用作类类的成员函数。



# 为了效率
# 为了安全
# 为了再类里访问自如

# 不需要函数调用的开销
# 唯一不同之处在于内联函数会在适当的地方像预定义宏一样展开


# 省去了函数调用时候的压栈，跳转，返回的开销


# **空间换时间**


# 任何在类内部定义的函数自动成为内联函数。




# 不能循环语句
# 不能过多判断
# 函数体不能过大
# 不能对函数进行取地址操作
